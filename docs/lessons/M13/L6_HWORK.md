## Задача 1 — Удалить значение X из вектора (in-place)

### Что нужно сделать

Из заданного пользователем вектора удалить все элементы равные X.

### Как работает программа

1. User вводит `n` — размер вектора
2. User вводит `n` целых чисел
3. User вводит `X` — удаляемое значение
4. **Удалить все** элементы == X из вектора
5. Вывести итоговое состояние

### Пример

```text
Input vector size: 5
Input numbers: 1 3 3 5 1
Input number to delete: 3
Result: 1 5 1
```

### Что оценивается

- Не удалено лишних элементов
- Все элементы == X удалены
- **Использована `pop_back()`** для удаления
- Порядок элементов не изменился
- Удаления **in-place**
- Решение использует **только один вектор**

### Подсказка к алгоритму

Constraint "только один вектор + pop_back + in-place" → классический паттерн **two-pointer**:

```cpp
int writePos = 0;
for (int i = 0; i < (int)vec.size(); i++) {
    if (vec[i] != X) {
        vec[writePos++] = vec[i];  // shift "хорошие" элементы влево
    }
}
// Теперь надо обрезать "хвост" — pop_back до writePos
while ((int)vec.size() > writePos) {
    vec.pop_back();
}
```

## Задача 2 — Вектор цен × вектор покупок (sum)

### Что нужно сделать

Вектор `prices` (float) — цены на продукты.
Вектор `items` (int) — индексы покупок.
Вычислить итоговую стоимость.

### Пример

```cpp
std::vector<float> prices {2.5, 4.25, 3.0, 10.0};
std::vector<int> items {1, 1, 0, 3};
// Покупки: индексы 1, 1, 0, 3 → 4.25 + 4.25 + 2.5 + 10.0 = 21.0
```

### Совет

> "При получении итоговой стоимости не нужно делать циклы по обоим векторам, достаточно одного цикла."

```cpp
float total = 0;
for (int idx : items) {
    total += prices[idx];
}
```

### Что оценивается

- Программа не допускает выхода за границы `prices`

## Задача 3 — Хранилище ≤20 элементов (ring buffer)

### Что нужно сделать

Embedded-системный store: **не больше 20 элементов** в векторе.

- При вводе нового элемента — добавить в store
- При вводе `-1` — вывести всё содержимое в **порядке добавления**
- Если store уже full — старые элементы удаляются (FIFO)

### Пример

```text
input: 1, 2, 3, ..., 19, 20  → output: 1 2 3 ... 19 20
input: 21, 22, 23 (overflow!) → -1 → output: 4 5 6 ... 19 20 21 22 23
```

При overflow первые 3 элемента (1, 2, 3) **затёрты** новыми (21, 22, 23).

### Реализация

Init вектора фиксированного размера:

```cpp
std::vector<int> db(20);  // сразу размер 20
```

### Совет (явная подсказка к алгоритму)

> [Кольцевой буфер](https://ru.wikipedia.org/wiki/Кольцевой_буфер). Этот алгоритм поможет быстрее найти решение.

Ring buffer pattern:

```cpp
std::vector<int> db(20);
int writePos = 0;     // куда писать следующий элемент
int totalCount = 0;   // сколько всего добавлено

void add(int v) {
    db[writePos] = v;
    writePos = (writePos + 1) % 20;
    totalCount++;
}

void print() {
    int start = (totalCount > 20) ? writePos : 0;
    int count = std::min(totalCount, 20);
    for (int i = 0; i < count; i++) {
        std::cout << db[(start + i) % 20] << " ";
    }
}
```

### Что оценивается

- Размер вектора **не превышает 20** элементов
- **Нет изменений** в размере вектора (`resize` НЕ должен вызываться) — только запись по индексам
