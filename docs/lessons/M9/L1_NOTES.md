## Task 9 — Маятник (pendulum)

### Условие

Пользователь вводит:

- начальная амплитуда колебаний (`start`)
- конечная амплитуда до которой должна уменьшиться (`finish`)

Каждое колебание уменьшает амплитуду на **8.4%** (т.е. умножение на `1 - 0.084 = 0.916`).
Найти количество колебаний.

### Алгоритм из урока

```cpp
#include <iostream>

int main() {
    double start, finish;
    std::cout << "Введите начальную и конечную амплитуду\n";
    std::cin >> start >> finish;

    // Контроль ввода
    while (start < 0 || finish < 0 || finish > start) {
        std::cout << "Некорректный ввод. Введите еще раз\n";
        std::cin >> start >> finish;
    }

    int count = 0;
    while (start > finish) {
        start *= (1 - 0.084f);
        count++;
    }

    std::cout << "Маятник достигнет нужной амплитуды через "
              << count << " колебаний\n";
    return 0;
}
```

### Тест-кейсы из видео

| start | finish | count |
|-------|--------|-------|
| 10    | 0.1    | 53    |
| 13    | 0.04   | 66    |

### Ключевые моменты

- Смешивание `int` и `float` в выражениях ОК — `int` неявно приводится к `float`
- При сравнении `float` с целым `0`, ноль автоматически приводится к `float`

### Соответствие в коде на диске

`Module8th/FinalTasksModule8/Pendulum/main.cpp` — реализация есть, **отличается**:

- использует `*= 0.916` (precomputed) vs урок `*= (1 - 0.084f)`
- обернуто в try/catch + кастомный `processUserInput()` (production-style)
- переменная названа `swings` вместо `count`

Ответы математически идентичны (`0.916 == 1 - 0.084`).

## Task 10 — Безопасная глубина для кладки яиц черепах

### Условие

Уровень опасности на глубине `x`:

```text
f(x) = x^3 - 3*x^2 - 12*x + 10
```

Пользователь вводит `level` — максимально допустимое отклонение `|f(x)|`.
Нужно найти такое `x`, что `|f(x)| <= level`.

### Алгоритм — бисекция (тот же подход что в quadratic root из M-1)

Известно:

- При `x = 0`: `f(0) = 10` (положительно)
- При `x = 4`: `f(4) = 64 - 48 - 48 + 10 = -22` (отрицательно)
- Где-то между — корень `f(x) = 0`

Шаги:

1. Завести `left = 0`, `right = 4`
2. `middle = (left + right) / 2`
3. Цикл while: `|f(middle)| > level`
   - Если `f(middle) > 0` → `left = middle` (двигаем левую границу)
   - Если `f(middle) < 0` → `right = middle`
   - Пересчитать `middle = (left + right) / 2`
4. Ответ — `middle`

### Реализация из урока

```cpp
#include <iostream>
#include <cmath>  // pow, fabs

int main() {
    double level;
    std::cout << "Введите допустимый уровень опасности\n";
    std::cin >> level;

    while (level < 0) {
        std::cout << "Некорректный ввод. Введите еще раз\n";
        std::cin >> level;
    }

    double left = 0, right = 4;
    double middle = (left + right) / 2;
    double result = 10;  // f(0) = 10, начальное значение

    while (std::fabs(result) > level) {
        result = std::pow(middle, 3) - 3 * std::pow(middle, 2)
               - 12 * middle + 10;
        if (result > 0) {
            left = middle;
        } else {
            right = middle;
        }
        middle = (left + right) / 2;
    }

    std::cout << "Глубина безопасной кладки равна " << middle << " метров\n";
    return 0;
}
```

### Pitfall

- Ответ — `middle` (точка глубины), а **не** `result` (значение функции)
- Цикл условие пишется **до** первого `result = ...` — иначе зайдёт сразу с `result = 10`,
  что больше любого разумного `level`. Вот почему инициализация `result = 10` важна.

### На диске

Реализации **нет** в `Module8th/`. Если понадобится — алгоритм бисекции выше готов
для копирования.

## Task 11 — Максимум двух чисел БЕЗ условий и циклов (optional)

### Идея

Вместо `if (a > b) max = a; else max = b;` использовать математику.

Пусть числа — точки на координатной прямой:

```text
middle = (a + b) / 2     // точка посередине
diff   = |a - b|          // расстояние между точками

max = middle + diff / 2   // двинуть от середины вправо на пол-расстояния
min = middle - diff / 2   // влево на пол-расстояния
```

### Реализация

```cpp
#include <iostream>
#include <cmath>  // fabs

int main() {
    double a, b;
    std::cin >> a >> b;

    double middle = (a + b) / 2;
    double diff   = std::fabs(a - b);

    double max = middle + diff / 2;
    double min = middle - diff / 2;

    std::cout << "Max = " << max << ", Min = " << min << "\n";
    return 0;
}
```

### Тест из видео

| a    | b    | max  | min  |
|------|------|------|------|
| 25   | 36   | 36   | 25   |
| -100 | -500 | -100 | -500 |

### Honest disclaimer (из видео)

> "На самом деле мы не совсем избавились от использования условных операторов.
> Дело в том, что мы использовали функцию `fabs`, которую реализовали не мы.
> Чтобы вычислить модуль числа, всё равно нужно проверять условие."

Преподаватель обещает что есть способ решить **полностью** без if/while/abs —
**но об этом позже** (в следующих модулях).

### На диске

Реализации **нет**. Идея с `(middle ± diff/2)` — полезный mental model для
"бранчлесс-программирования" в SIMD/GPU контексте.

## Take-aways

1. **Bisection method** (binary search для уравнений) — полезный паттерн,
   тот же подход применим к `sqrt`, `cbrt`, поиску корней любого монотонного выражения.
2. **`int + float` mixed expressions** — компилятор promote к `float` автоматически,
   `0` сравнивается с `float` корректно.
3. **`<cmath>`** функции — `pow`, `fabs`, `abs` (для int).
4. **"Branchless math"** — иногда `if` можно заменить на формулу с `abs`.
