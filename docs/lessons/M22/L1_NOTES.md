# M22.1 — Введение в словари (std::map)

**Длительность:** ~8 минут
**Тема:** концепция словаря, `std::map`, операции `[]`, важное правило уникальности ключей

## Главная идея

> `std::map<K, V>` = ассоциативный упорядоченный контейнер «ключ → значение».
> Ключ уникален; быстрый поиск по ключу — основное преимущество.

## Зачем нужен новый контейнер

`std::vector` — последовательный массив, доступ по числовому индексу.
Подходит, когда:
- Данные вводятся последовательно
- Доступ — по позиции

Но что если нужно «найти жильца по номеру квартиры»? Vector подходит плохо
(линейный поиск). Здесь нужен **словарь** — структура для быстрого lookup
по произвольному ключу.

## Аналогия: толковый словарь

| Толковый словарь   | std::map                |
|--------------------|-------------------------|
| слово              | ключ (key)              |
| определение        | значение (value)        |
| алфавитный порядок | внутренний порядок K    |
| быстрый поиск      | поиск по ключу O(log N) |

## Определение std::map

> `std::map` — **ассоциативный упорядоченный** контейнер.

### Ассоциативный

Хранит **пары** `(key, value)`. Ключ используется для упорядочивания и поиска,
значение — собственно полезная информация.

### Упорядоченный

Контейнер сам поддерживает порядок ключей. Не за счёт перестановки массива
(дорого), а специальной структурой данных (внутри — обычно red-black tree,
тема следующих уроков).

## Создание std::map

```cpp
#include <map>
#include <string>

std::map<int, std::string> apartPerson;
//        ^^^  ^^^^^^^^^^
//        key  value
//        type type
```

**Шаблонные параметры в `<>`** через запятую: тип ключа, тип значения.
Похоже на `vector<T>`, только два параметра.

## Доступ через `[]`

### Запись

```cpp
apartPerson[2] = "Иванов";   // квартира 2 → Иванов
apartPerson[5] = "Петров";   // квартира 5 → Петров
```

### Чтение

```cpp
std::cout << apartPerson[5] << "\n";   // Петров
```

## Подводный камень: `[]` создаёт элемент

```cpp
std::cout << apartPerson[4] << "\n";   // выведет пустую строку
```

**Что произошло:** оператор `[]` для несуществующего ключа **создаёт новый элемент**
с дефолтным значением (`std::string{}` = пустая строка). Контейнер теперь
содержит запись `4 → ""`.

**Опасность:** если value-тип не имеет дефолтного значения или операция дорогая,
случайное обращение портит данные / производительность.

**Правильный поиск:** через `.find()` или `.count()` (тема позже):
```cpp
auto it = apartPerson.find(4);
if (it != apartPerson.end()) std::cout << it->second;
```

## Уникальность ключей

> В `std::map` ключи **уникальны** — один ключ хранит **одно** значение.

При повторной записи по существующему ключу — старое значение **затирается**:

```cpp
apartPerson[5] = "Петров";    // 5 → Петров
apartPerson[5] = "Сидоров";   // 5 → Сидоров (Петров удалён)
std::cout << apartPerson[5];  // Сидоров
```

**Аналогия учителя:** ЖКХ — в одной квартире может жить только один прописанный
(в простой модели). Новый жилец вытесняет старого.

## Сравнение с std::vector

| Аспект          | std::vector       | std::map                |
|-----------------|-------------------|-------------------------|
| Доступ          | по индексу `int`  | по ключу любого типа    |
| Порядок         | вставки           | автоматически по K      |
| Поиск           | O(N) линейный     | O(log N) дерево         |
| Дубликаты       | разрешены         | ключи уникальны         |
| `[]` для нового | UB / out_of_range | создаёт `K → V{}`       |
| Память          | непрерывный блок  | дерево узлов            |

## Practical relevance для DevOps

- **Configs:** `map<string, string>` — пары `key=value` из ini/env
- **Метрики:** `map<string, int>` — счётчики событий по типам
- **Кэш:** key → cached result
- **Roster:** `map<id, ServerInfo>` — быстрый lookup сервера по ID
- **Дедупликация:** `map<string, bool>` для уникальных ключей
