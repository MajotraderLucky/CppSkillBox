## 1. Телефонный справочник

### Что нужно сделать

На вход 3 типа запросов:

| Запрос                  | Действие                                  |
|-------------------------|-------------------------------------------|
| `69-70-30 Ivanov`       | добавить (телефон → фамилия)              |
| `69-70-30`              | по номеру → фамилия                       |
| `Ivanov`                | по фамилии → ВСЕ её телефоны через пробел |

### Особенности

- **Телефон уникален** (он ключ)
- **Фамилия НЕ уникальна** — у одной фамилии может быть много телефонов

### Сложность

`O(log N)` для запросов 1, 2.
`O(log N + количество_номеров_с_фамилией)` для запроса 3.

### Подсказки по реализации

- **Прямой словарь:** `map<phone, name>` — для запросов 1, 2
- **Обратный индекс:** `map<name, vector<phone>>` или `multimap<name, phone>` — для запроса 3
- При добавлении обновляются ОБА словаря

### Различение запросов

Если строка содержит `-` (или начинается с цифры) → это телефон.
Иначе — фамилия.

## 2. Регистратура

### Что нужно сделать

Очередь пациентов. На вход:
- **Фамилия** — встать в очередь
- **`Next`** — вызвать первого по **лексикографическому порядку** (а-z), вывести и удалить

### Особенности

- **Фамилии могут повторяться** — нужно учитывать count
- **Lexicographic order** — `Ivanov` < `Petrov` < `Sidorov`

### Сложность

Каждая операция (add / Next) за **O(log N)**.

### Пример

```text
in:  Sidorov
in:  Ivanov
in:  Ivanov
in:  Petrov
in:  Next
out: Ivanov
in:  Next
out: Ivanov
in:  Next
out: Petrov
```

### Подсказки по реализации

- `map<surname, count>` — отсортированные ключи + счётчик дубликатов
- Для `Next`: `m.begin()` даёт минимум за O(1)
- Декрементировать count, при count==0 — `erase(begin())`

(Можно использовать `multiset`, но spec ограничивает заголовки `<map>`.)

## 3. Анаграммы

### Что нужно сделать

Функция:

```cpp
bool isAnagram(const std::string& a, const std::string& b);
```

Возвращает `true` если строки **анаграммы** (одни и те же символы, разный порядок).

**Пример:** `лекарство` ↔ `стекловар` → true

### Чек-лист

- Принимает корректные типы данных (две строки)
- Возвращаемый тип — `bool`
- Использует только `<iostream>`, `<map>`, `<string>`
- Корректно определяет анаграммы

### Подсказки по реализации

- Подсчитать частоту каждой буквы в каждой строке: `map<char, int>`
- Сравнить два словаря (operator== работает)
- Сложность: O((N+M) log K), где K = размер алфавита

### Edge cases

- Разная длина → точно не анаграмма
- Пустые строки → анаграммы (обе пустые)
- Регистр — по умолчанию case-sensitive (учитель не уточняет, делаем case-sensitive)
