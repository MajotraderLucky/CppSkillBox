# M22.4 — Итерация по словарю и удаление элементов

**Длительность:** ~9 минут
**Тема:** `begin()`, `end()`, итерация в for-цикле, `erase()`

## Главная идея

> Итерация по `std::map` через `[begin, end)` даёт элементы **в порядке возрастания ключа**.
> `erase` принимает итератор или ключ.

## Итераторы begin / end

```cpp
std::map<int, std::string> omap;
omap.insert(std::make_pair(10, "10"));
omap.insert(std::make_pair(11, "11"));
omap.insert(std::make_pair(8,  "8"));
omap.insert(std::make_pair(6,  "6"));
omap.insert(std::make_pair(7,  "7"));
omap.insert(std::make_pair(12, "12"));
omap.insert(std::make_pair(2,  "2"));
```

| Итератор    | Указывает на                                   |
|-------------|------------------------------------------------|
| `begin()`   | первый элемент (минимальный ключ)              |
| `end()`     | **за** последним (sentinel, не разыменовывать) |

**Полуоткрытый интервал** `[begin, end)` — стандартный pattern STL.
Цикл от begin до end проходит все элементы, не выходя за границы.

## Цикл for по map

```cpp
for (std::map<int, std::string>::iterator it = omap.begin();
     it != omap.end();
     ++it) {
    std::cout << it->first << " " << it->second << "\n";
}
```

### Современный C++ (короче)

```cpp
for (auto it = omap.begin(); it != omap.end(); ++it) {
    std::cout << it->first << " " << it->second << "\n";
}

// Или ещё короче (range-based for):
for (const auto& [key, value] : omap) {
    std::cout << key << " " << value << "\n";
}
```

## Гарантия порядка обхода

Хоть мы вставляли в случайном порядке (10, 11, 8, 6, 7, 12, 2),
итерация даёт **отсортированный** результат:

```text
2 "2"
6 "6"
7 "7"
8 "8"
10 "10"
11 "11"
12 "12"
```

**Почему:** map хранится в RB-tree (см. M22.2). При in-order обходе
дерева поиска получаем элементы в порядке возрастания ключа.

## Поиск минимума за O(1) после построения

```cpp
std::cout << omap.begin()->first << "\n";   // 2 (минимальный ключ)
```

Не нужен `std::min_element` — `begin()` уже указывает на минимум.

Аналогично последний элемент:
```cpp
auto it = --omap.end();   // step back from sentinel
std::cout << it->first;   // максимум
```

## Удаление через erase

### Способ 1: по итератору

```cpp
auto itf = omap.find(10);
omap.erase(itf);    // удалить элемент с ключом 10
```

### Способ 2: по ключу

```cpp
omap.erase(7);      // удалить элемент с ключом 7
```

`erase` по ключу возвращает количество удалённых (0 или 1 для map).

### Когда использовать что

| Ситуация                              | Способ      |
|---------------------------------------|-------------|
| Уже есть итератор после find          | `erase(it)` |
| Просто знаем ключ                     | `erase(k)`  |
| Удаление в цикле обхода               | `erase(it)` |

### Удаление во время итерации

```cpp
for (auto it = omap.begin(); it != omap.end(); ) {
    if (нужно_удалить(it)) {
        it = omap.erase(it);   // erase возвращает следующий итератор
    } else {
        ++it;
    }
}
```

**Важно:** не делать `erase(it)` + `++it` — итератор инвалидируется
после erase. Используем `it = erase(it)` (since C++11).

## Свойства итераторов map

- **Bidirectional:** есть `++` и `--`, нет `+` / `-` / `<`
- **Стабильны:** insert/erase других элементов не инвалидируют итераторы
- **erase инвалидирует** только итератор на удалённый элемент
- **Доступ через `->`** или `*it` (даёт `pair&`)

## Сравнение итераторов разных контейнеров

| Контейнер       | Категория итератора | Операторы              |
|-----------------|---------------------|------------------------|
| `vector`        | random-access       | `++ -- + - < > [n]`    |
| `list`          | bidirectional       | `++ --`                |
| `map` / `set`   | bidirectional       | `++ --`                |
| `forward_list`  | forward             | `++` только            |

## Practical relevance для DevOps

- **Top-N запросов:** range-for по map<count, query>, останов по threshold
- **Очистка устаревших:** erase в цикле по timestamp ключу
- **Audit log:** обход session_id → action в порядке времени
- **Sorted output:** map даёт sorted list бесплатно — без std::sort
