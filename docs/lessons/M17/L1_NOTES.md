# M17.1 — Что такое указатель

**Длительность:** ~12 минут
**Тема:** введение в указатели (pointers), `&` (address-of), `*` (dereference)

## Главная идея

> Указатель — это объект, который хранит **адрес** другой переменной в памяти.

## Базовые операции

```cpp
int i = 10;
int* pi = &i;       // pi хранит адрес i
std::cout << pi;    // печатает адрес (hex)
std::cout << *pi;   // печатает 10 (разыменование)
*pi = 20;           // меняет значение по адресу → теперь i = 20
```

### `&` (амперсанд) = address-of

`&variable` возвращает адрес переменной в hex (например `0x7ffd1234abcd`).

Контраст с другими ролями `&`:
- `bool flag = a & b` — побитовое И
- `if (a && b)` — логическое И
- `int& ref = i` — ссылка (объяснит позже)

### `*` (звёздочка) = dereference

Перед указателем — получает значение по адресу:
- `*pi` (rvalue) — читает: `int x = *pi;`
- `*pi = 20` (lvalue) — пишет

Перед типом — объявляет указатель: `int* pi`.

## Типы указателей

```cpp
int* pi;       // указатель на int
float* pf;     // указатель на float
double* pd;    // указатель на double
char* pc;      // указатель на char
```

Тип нужен, чтобы:
- Знать, сколько байт читать/писать через `*pi`
- Знать арифметику указателей (M17.2 покажет)

## Адреса меняются между запусками

При каждом запуске программы — переменные могут лежать в разных адресах
(ASLR, компилятор, OS). Это **нормально**. Главное — относительные операции
работают одинаково.

## Перенаправление указателя

```cpp
int a = 10, b = 5;
int* pa = &a;
pa = &b;         // теперь pa указывает на b!
*pa = 20;        // меняем b → 20 (а НЕ a)
```

Указатель — обычная переменная. Можно **переприсвоить** ему другой адрес.

## Practical relevance для DevOps

- **C arrays / strings:** char* как указатель на первый элемент
- **Linux kernel APIs:** все callbacks через function pointers
- **System programming:** mmap, malloc возвращают указатели
- **Python C extensions:** PyObject* — везде указатели
- **gdb debugging:** print *ptr, watch *ptr, ptype ptr
