# M35.2 — Стандарты C++ (комитет, версии, поддержка компиляторов)

**Длительность:** ~10 минут
**Тема:** ISO C++ standardization, история языка, legacy vs modern, fragmentation

## Главная идея

> **C++** имеет **две природы**: современные удобные инструменты (`std::map`,
> `std::vector`) живут поверх **низкоуровневого C-наследия** (raw указатели,
> сырые массивы). Стандарт развивает язык, но компиляторы поддерживают его
> **разрозненно**.

## Современные vs Legacy инструменты

| Инструмент                 | Эпоха       | Безопасность            | Пример                        |
|----------------------------|-------------|-------------------------|-------------------------------|
| `std::map<K,V>`            | Современный | Защита от ошибок        | Богатый API                   |
| `std::vector<T>`           | Современный | Bounds checking (`.at`) | resize, emplace               |
| Raw `T*` указатели         | Legacy      | Нет                     | Утечки, double-delete         |
| C-style массивы `int a[N]` | Legacy      | Нет                     | Buffer overflow               |

> «Это **инструменты из двух разных эпох или даже языков**.»
>
> **Modern C++** — это **наслоение** новых возможностей поверх старого C-стиля 90-х.

## Цитата от создателей

> «Язык, который сегодня называется C++, и каким он был в 90-е, — это **два разных языка**. Писать на них нужно по-разному.»

## Legacy code

> **Большие проекты пишутся **долго**.** Раньше — один способ. Сейчас — другой.
> Разработчику приходится **уметь оба**.

## Аналогия: мобильные ОС

| Аспект                        | Мобильные ОС                     | C++                                |
|-------------------------------|----------------------------------|------------------------------------|
| Версии                        | Android 12 / iOS 17              | C++11 / 17 / 20 / 23               |
| Каждая новая версия           | Новые API                        | Новые языковые возможности         |
| Указание min version          | Compile-time check               | `-std=c++17`                       |
| Кто диктует                   | Google / Apple                   | ISO Committee                      |

> Разработчик мобильного приложения говорит: «нужен голосовой ввод» → IDE поднимает min OS.
> Разработчик C++ говорит: «нужны concepts» → требует C++20.

## ISO C++ Committee

> **Комитет по стандартизации C++** (ISO/IEC JTC1/SC22/WG21).

### Как работает

| Шаг                            | Кто                                            |
|--------------------------------|------------------------------------------------|
| Раз в 3 года                   | Выпускают **новый стандарт** (документ ISO)    |
| Документ                       | Список **всех** возможностей (не только новых) |
| Имена                          | По году: **C++11, C++14, C++17, C++20, C++23** |
| Реализация                     | **Разработчики компиляторов** — отдельно       |

### Versions история

| Год  | Имя       | Ключевые фичи                                      |
|------|-----------|----------------------------------------------------|
| 1998 | C++98     | Первый стандарт ISO                                |
| 2003 | C++03     | Bug fixes для 98                                   |
| 2011 | C++11     | auto, nullptr, lambdas, move semantics, smart_ptr  |
| 2014 | C++14     | Generic lambdas, return type deduction             |
| 2017 | C++17     | optional, variant, structured bindings, filesystem |
| 2020 | C++20     | Concepts, ranges, modules, coroutines              |
| 2023 | C++23     | std::expected, std::print, deducing this           |

## Fragmentation: компиляторы поддерживают разрозненно

> Главное отличие от мобильных ОС: **комитет не контролирует** ни компиляторы, ни производителей.

### Цепочка реализации

```text
ISO Committee (документ-стандарт)
        │ читает
        ▼
Разработчики компиляторов (gcc / clang / MSVC / Intel)
        │ реализуют
        ▼
Версии компиляторов с ЧАСТИЧНОЙ поддержкой стандарта
        │ используют
        ▼
Разработчики приложений
```

> Компилятор может **выбрать** какие фичи реализовать, а от каких **отказаться** (cost vs benefit).

### Текущее состояние (примерно)

| Компилятор | C++11 | C++14 | C++17 | C++20      | C++23      |
|------------|-------|-------|-------|------------|------------|
| GCC        | 100%  | 100%  | 100%  | ~99%       | partial    |
| Clang      | 100%  | 100%  | 100%  | ~95%       | partial    |
| MSVC       | 100%  | 100%  | ~99%  | ~95%       | partial    |

> Точные данные — `cppreference.com/compiler_support`.

## Practical implication

> **Если ты пишешь cross-platform приложение** — не гонись за последним стандартом.
>
> Изучи, **что поддерживают** твои целевые платформы.

### Где смотреть

- **cppreference.com** — таблица compiler support
- **Compiler Explorer (godbolt.org)** — попробовать компиляцию на разных компиляторах
- **Issue tracker** компилятора — что не реализовано

## C++ как язык

> «C++ — относительно **разрозненный** инструмент. Впитал множество идей.»

### Аналогия со строительством

| Язык           | Уровень контроля                               |
|----------------|------------------------------------------------|
| Java / Python  | «Готовые проекты домов» — меняешь параметры    |
| C++ modern     | «Готовый проект» **+** при необходимости       |
| C++ legacy / C | Можно изменить **состав вещества кирпича**     |

> **Цена** низкоуровневого контроля = **сложность** проекта.

## Modern C++ ≈ Python в типичных задачах

> «В задачах на практике современный код на C++ очень сильно похож на Python.»

```cpp
// Modern C++: STL + algorithms + lambdas + ranges
auto evens = numbers
    | std::views::filter([](int n) { return n % 2 == 0; })
    | std::views::transform([](int n) { return n * 2; });
```

vs

```python
# Python
evens = [n * 2 for n in numbers if n % 2 == 0]
```

## Стандарт vs реальность

> Урок «полон теории» — потому что комитет занимается **скучными** документами.
> Но эти документы **напрямую** определяют облик языка через 3 года.

## Что дальше в модуле

> Разбор **наиболее часто используемых** modern C++ фич, которые:
> - Сильно повлияли на облик языка
> - Облегчают каждодневные задачи

Ожидаются в M35.3+:
- `auto` / type deduction
- Range-based for
- Lambdas
- Structured bindings (C++17)
- `std::optional`, `std::variant` (C++17)
- `if constexpr` (C++17)
- Concepts (C++20)
- Возможно: ranges, coroutines (C++20)

## Practical relevance для DevOps

- **CI matrix:** тестировать на нескольких компиляторах (gcc + clang) — найти incompatibilities
- **`-std=c++17` явно** в CMakeLists.txt — не полагаться на default
- **Compiler upgrade:** пересобирать всё с новой версией перед deployment
- **Static analysis:** clang-tidy подсказывает modern C++ замены legacy кода
- **Code review:** новый код — modern style; legacy — оставлять как есть, не рефакторить ради рефакторинга
