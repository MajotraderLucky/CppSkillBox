## Эталонное решение учителя

### Структура данных

- **5 номиналов:** 100, 200, 500, 1000, 5000 (`bills[]` для random pick)
  — учитель в видео пропустил 2000, хотя оно есть в spec
- **Массив 1000 ячеек** — каждая ячейка хранит сам номинал купюры либо 0
  (отсутствующая позиция)
- Файл `money.bin` хранит весь массив 1000 элементов (~4000 bytes)

### Workflow

1. Прочитать команду `operation` из stdin (single char)
2. `is_open()` для money.bin → load existing OR init zeros
3. **`+`:** проход по массиву, для каждого 0 → `bills[rand()%5]`,
   запись в money.bin
4. **`-`:** считать `amount`, проверить `amount % 100 == 0`,
   `amountToCollect = amount`, проход по убыванию номиналов
   (5000→100), для каждой подходящей купюры — `amountToCollect -= bill;
   ячейка = 0`. Если в конце `amountToCollect != 0` → ошибка
5. **Завершение:** запись массива в money.bin
6. Программа завершает работу после ОДНОЙ операции (не цикл)

## Сравнение моего решения с эталоном

### Структура данных

| Аспект            | Учитель                          | Моё (Task4_ATM)                    |
|-------------------|----------------------------------|------------------------------------|
| Номиналы          | 5: 100/200/500/1000/5000         | 6: 100/200/500/1000/2000/5000      |
| Storage           | int[1000] (сами купюры или 0)    | struct{int counts[6]} (счётчики)   |
| Размер state file | ~4000 bytes                      | 24 bytes                           |
| Fill cost         | O(N=1000) loop                   | O(K=6) для fill через rand counts  |

**Я следую полному spec:** «100, 200, 500, 1000, 2000 и 5000» — там
**6 видов** включая 2000. Учитель забыл 2000 в видео.

**Storage approach:** мой через counts[] компактнее (24B vs 4000B) и
быстрее, но менее буквально следует «массив всех купюр». Оба валидны.

### Workflow

| Аспект              | Учитель                          | Моё                                  |
|---------------------|----------------------------------|--------------------------------------|
| Iteration           | Одна операция → выход            | Цикл commands (+/-/q)                |
| Commands            | + или - (single)                 | +, -, q                              |
| Withdrawal greedy   | Цикл по 1000 купюрам, проверка   | Цикл по 6 номиналам с min(count, n)  |
| Persistence         | Save после каждой операции       | Save при выходе (q)                  |

**Учитель явно требует:** «После выполнения любой из операций, программа
завершает свое выполнение» — мой command-loop отступает от этого.

**Greedy implementation:** мой O(K=6) греди использует
`needed[i] = min(counts[i], left/DENOMS[i])` — математически
эквивалентен учительскому проходу по массиву купюр.
