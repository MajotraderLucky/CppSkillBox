## 2. Модель данных для посёлка

**Что делаем:** спроектировать иерархию структур для посёлка.

### Сущности

- **Посёлок** = массив участков
- **Участок** имеет номер + 0..N построек
- **Постройки:** жилой дом, гараж, сарай, баня (у каждой — площадь)
- **Жилой дом:** 1-3 этажа
- **Этаж:** 2-4 комнаты + высота потолка
- **Комната:** тип (`enum`: спальня, кухня, ванная, детская, гостиная) + площадь
- **Печь с трубой** может быть в доме и бане

### Требования

- Структуры **связные и легко расширяемые**
- **Без дублирующих** конструкций
- Реализовать **заполнение** (cin) через цикл
- Рассчитать **суммарный параметр**: например процент площади построек от общей

### Подсказки

- Применять `enum` для типов комнат и зданий
- Вложенные структуры
- `vector<T>::push_back` для динамического добавления

## 3. Математический вектор

**Что делаем:** структура 2D-вектора + операции.

### Структура

```cpp
struct Vec2 {
    double x;
    double y;
};
```

### Команды (отдельные функции)

- **`add`**       — сумма двух векторов → вектор
- **`subtract`**  — разность двух векторов → вектор
- **`scale`**     — умножение вектора на скаляр → вектор
- **`length`**    — длина вектора → скаляр (sqrt(x² + y²))
- **`normalize`** — нормализация вектора → единичный вектор того же направления

### Workflow

1. Пользователь вводит команду
2. В зависимости от команды — вводит операнды (векторы и/или скаляры)
3. Результат в stdout

## 4. Пошаговая ролевая игра (необязательная)

**Что делаем:** упрощённый turn-based RPG.

### Карта

- Размер **20×20**
- Игрок: `P`, противники: `E`, пустые клетки: `.`
- Карта показывается **после каждого хода игрока**

### Структура персонажа

```cpp
struct Character {
    std::string name;
    int health;
    int armor;
    int damage;
    Vec2i position;   // x, y координаты
    bool is_player;   // флаг
};
```

### Initial setup

- **5 противников** в случайных клетках:
  - Имя: `"Enemy #N"`
  - HP: `[50, 150]` random
  - Armor: `[0, 50]` random
  - Damage: `[15, 30]` random
- **Игрок:** все параметры задаёт пользователь, позиция случайная

### Команды хода игрока

- `L`, `R`, `U`, `D` — left/right/up/down
- `save` — сериализовать состояние в `save.bin`
- `load` — восстановить состояние из `save.bin` (если файл существует)

### Правила хода

- Если ход в клетку с другим персонажем → **наносит ему урон** (не двигается)
- Противники ходят в **случайном направлении**
- Противники **не атакуют друг друга**
- За пределы карты ходить нельзя — **ход пропускается**

### Урон (формула из M21.2)

```cpp
target.armor  -= damage;
target.health += target.armor < 0 ? target.armor : 0;
target.armor   = max(0, target.armor);
```

### Конец игры

- **Все противники мертвы** → "Победа"
- **Игрок мёртв** → "Поражение"

### Save/Load

- Бинарный формат
- Соблюдать **последовательность полей** при write/read
- **Проверять существование файла** перед load
- Сохранять/загружать имена корректно (length-prefix как в M21.4)
