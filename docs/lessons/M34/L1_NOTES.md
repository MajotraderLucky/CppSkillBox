# M34.1 — Двумерная графика и буфер экрана

**Длительность:** ~15 минут
**Тема:** пиксель, RGB модель, framebuffer, GPU/shaders, графические API

## Главная идея

> Экран = массив **пикселей**. Каждый **пиксель** = данные о цвете (RGB + alpha).
> Изображение = `Framebuffer` (массив пикселей).
> Современная графика рисуется через **GPU** + **API** (OpenGL/Vulkan/DirectX/Metal).

## Что такое пиксель

> **Pixel** = от **Pic**ture **El**ement (элемент изображения).
>
> Это единица данных, кодирующая цвет точки экрана. **Просто байты, нули и единицы.**

## RGB модель

| Канал | Цвет        | Диапазон  |
|-------|-------------|-----------|
| R     | Red         | 0–255     |
| G     | Green       | 0–255     |
| B     | Blue        | 0–255     |

- 1 байт на канал → `unsigned char` (или `char`)
- 3 канала → 256³ = **~16.7 млн** различных цветов

### Структура Color в C++

```cpp
struct Color {
    unsigned char r;
    unsigned char g;
    unsigned char b;
};
```

> Пиксель — это вектор `(r, g, b)` в 3D пространстве цветов, координаты вместо `x, y, z`.

## Аддитивная vs субтрактивная модель

| Модель                   | Где                        | Логика                             |
|--------------------------|----------------------------|------------------------------------|
| **Аддитивная** (RGB)     | Мониторы, диоды, проекторы | Складываем световые компоненты     |
| **Субтрактивная** (CMYK) | Краски, печать             | Вычитаем (пигменты поглощают свет) |

> Краски: `синий + жёлтый = зелёный`. Все вместе → грязный чёрный.
> Свет: все RGB по 255 → **белый**. Все по 0 → **чёрный**.

### Примеры цветов

| Цвет             | R   | G   | B   |
|------------------|-----|-----|-----|
| Красный          | 255 | 0   | 0   |
| Зелёный          | 0   | 255 | 0   |
| Синий            | 0   | 0   | 255 |
| Белый            | 255 | 255 | 255 |
| Чёрный           | 0   | 0   | 0   |
| Жёлтый           | 255 | 255 | 0   |
| Розовый          | 255 | 0   | 255 |
| Циан (бирюзовый) | 0   | 255 | 255 |

## Alpha-канал (прозрачность)

> **Alpha (A)** = **opacity** или **transparency**.
> 4-й параметр пикселя для **смешивания слоёв**.

```cpp
struct Color { unsigned char r, g, b, a; };   // RGBA
```

- Не отображается монитором напрямую (мониторы непрозрачные)
- **Растеризатор** / графическая библиотека использует alpha для **компоновки слоёв**
- Полупрозрачное окно поверх фона — alpha blending

## Другие форматы цвета

| Формат              | Биты на канал   | Применение             |
|---------------------|-----------------|------------------------|
| RGB / RGBA 8-bit    | 8               | Стандарт мониторов     |
| HDR 10-bit          | 10              | Расширенный диапазон   |
| Float (RAW)         | 32 (float)      | Профессиональное фото  |
| Grayscale           | 1 канал         | Чёрно-белое            |
| Indexed (palette)   | 1 байт = индекс | GIF, retro             |

> Для базовых задач (UI, web) — 8-bit RGBA достаточно.

## Framebuffer

> **Framebuffer** = массив всех пикселей экрана. Память, которую видеокарта **сканирует** для отображения.

### Размер для Full HD (1920×1080)

```text
1920 × 1080 = 2 073 600 пикселей
× 4 байта (RGBA) = ~8 МБ на кадр
```

### В C++ (концептуально)

```cpp
const int WIDTH  = 1920;
const int HEIGHT = 1080;
Color framebuffer[WIDTH * HEIGHT];
```

> На практике framebuffer **управляется драйвером видеокарты** через **API**, не напрямую массивом.

## Видеокарта и GPU

### Зачем GPU

| CPU                    | GPU                                  |
|------------------------|--------------------------------------|
| General-purpose        | Specialized for parallel pixel ops   |
| Несколько ядер (4–32)  | Тысячи параллельных ядер             |
| Сложная инструкция     | Простая, массовая                    |
| Многоскалярные ops     | **Сверхмногоскалярные** ops          |

> Для CPU перемножить два числа — норма. Для GPU **нарисовать треугольник** — норма.

### Шейдеры

> **Shader** = микропрограмма, выполняющаяся **на GPU** для каждого пикселя/фрагмента.

| Тип             | Назначение                                |
|-----------------|-------------------------------------------|
| Vertex shader   | Преобразование вершин в координаты экрана |
| Fragment shader | Цвет каждого пикселя                      |
| Geometry shader | Создание новых вершин                     |
| Compute shader  | Вычисления общего назначения на GPU       |

> Эффекты: размытие (blur), рефракция, тени, освещение — всё через шейдеры.

## Графические API

> Никто не пишет под видеокарту **напрямую** — слишком много моделей, разные инструкции.
> Поэтому существуют **API** = унифицированный слой.

| API        | Платформы               | Кто разработал             |
|------------|-------------------------|----------------------------|
| OpenGL     | Cross-platform          | Khronos (исторически SGI)  |
| Vulkan     | Cross-platform (modern) | Khronos                    |
| DirectX    | Windows                 | Microsoft                  |
| Metal      | macOS / iOS             | Apple                      |
| WebGL      | Браузеры                | Khronos (на основе OpenGL) |

> **Qt** (которая будет дальше в курсе) под капотом использует **OpenGL**.

### Зачем абстракция

- Приложение пишется один раз → работает на разном железе
- Производитель видеокарты реализует API в драйвере → совместимость
- Стандарт обновляется, но **API API остаётся стабильным**

## Связь с архитектурой

```text
Application (твоя программа)
    │ uses
    ▼
Graphics API (OpenGL, Vulkan, DirectX)
    │ implemented by
    ▼
Driver (NVIDIA, AMD, Intel)
    │ talks to
    ▼
GPU (hardware)
    │ writes to
    ▼
Framebuffer (memory)
    │ scanned by
    ▼
Display
```

## Practical relevance для DevOps

- **Headless servers:** обычно нет GPU → есть **virtual framebuffer** (Xvfb) для тестирования GUI
- **Container GUI:** прокидывание X11 socket / VNC / RDP
- **GPU passthrough:** виртуализация GPU (NVIDIA vGPU, MIG) для ML / rendering
- **Nvidia drivers + CUDA:** для ML вычислений на GPU
- **Wayland vs X11:** новый display protocol vs legacy
- **GPU мониторинг:** `nvidia-smi`, `radeontop`, `intel_gpu_top`
