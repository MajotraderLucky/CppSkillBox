# M23.2 — Макросы с аргументами

**Длительность:** ~8 минут
**Тема:** функциеподобные макросы, обобщённые min/max, опасность двойного вычисления

## Главная идея

> Функциеподобный макрос = текстовая функция без типов.
> Один макрос работает с любыми типами, но опасен **двойным вычислением аргументов**.

## Зачем макросы вместо функций

Задача: `max(a, b)` для **любых типов** (int, double, ...).
С обычными функциями — нужны перегрузки или templates:

```cpp
int max(int a, int b)         { return a > b ? a : b; }
double max(double a, double b){ return a > b ? a : b; }
// ... и так для каждого типа
```

С макросом — **одна** реализация:

```cpp
#define MAX(a, b) ((a) > (b) ? (a) : (b))
```

## Объявление функциеподобного макроса

```cpp
#define MIN(a, b) ((a) < (b) ? (a) : (b))
//      ^^^^^^^^   ^^^^^^^^^^^^^^^^^^^^^^^^
//      имя+args   определение (тело)
```

**Особенности:**
- Открывающая скобка СРАЗУ после имени (без пробела!) — иначе препроцессор
  сочтёт `(a, b)` частью значения, а не списком параметров
- В аргументах **нет типов** — препроцессор не знает о типах
- В определении — любой код: операторы, скобки, вызовы функций

### Пример: MIN и MAX

```cpp
#define MIN(a, b) ((a) < (b) ? (a) : (b))
#define MAX(a, b) ((a) > (b) ? (a) : (b))

int main() {
    std::cout << MAX(5, 10);          // 10
    std::cout << MAX(MIN(30, 50), 10); // 30 (вложенные!)

    double pi = 3.14;
    std::cout << MAX(pi, 33.3);       // 33.3 (с double тоже работает)
}
```

## ОЧЕНЬ важно: скобки везде

```cpp
#define BAD_MAX(a, b)  a > b ? a : b               // ← BAD
#define GOOD_MAX(a, b) ((a) > (b) ? (a) : (b))     // ← GOOD
```

**Почему скобки нужны:**

```cpp
#define SQR(x) x * x

int y = SQR(2 + 3);
// раскрывается: 2 + 3 * 2 + 3 = 2 + 6 + 3 = 11   (НЕ 25!)

#define SQR(x) ((x) * (x))
int y = SQR(2 + 3);
// раскрывается: ((2 + 3) * (2 + 3)) = 25  ← OK
```

> **Правило:** каждый аргумент в макросе обернуть в `()`,
> весь результат тоже в `()`.

## ОПАСНОСТЬ: двойное вычисление аргументов

Препроцессор подставляет аргумент **дважды** если он встречается в теле:

```cpp
#define MAX(a, b) ((a) > (b) ? (a) : (b))

double r = MAX(pow(2, 4), 10);
// раскрывается:
double r = ((pow(2, 4)) > (10) ? (pow(2, 4)) : (10));
//          ^^^^^^^^^^^                   ^^^^^^^^^^^
//          ВЫЗОВ #1                       ВЫЗОВ #2
```

**Что плохо:**
- `pow()` вычисляется ДВАЖДЫ — в 2 раза медленнее
- Если функция с **side effects** — последствия непредсказуемы

### Особенно опасно с side effects

```cpp
#define MAX(a, b) ((a) > (b) ? (a) : (b))

int i = 5;
int x = MAX(++i, 10);
// раскрывается: ((++i) > (10) ? (++i) : (10))
// если ++i > 10 (i стало 6, false) → выберет 10
// но i уже инкрементнут ОДИН раз
// в другом случае — мог быть инкрементнут ДВА раза
```

Поведение зависит от ветки тернарного оператора → bug.

### Современная альтернатива

```cpp
#include <algorithm>
auto r = std::max(pow(2, 4), 10.0);   // template, безопасно
```

С C++ templates — тот же эффект «работает с любым типом», без подводных камней.

## Сравнение макрос vs функция vs template

| Аспект                  | #define MAX | int max(int,int) | template max |
|-------------------------|-------------|------------------|--------------|
| Универсальность типов   | да          | нет              | да           |
| Type checking           | нет         | да               | да           |
| Двойное вычисление      | возможно    | нет              | нет          |
| Side effects safe       | НЕТ         | да               | да           |
| Виден в дебаггере       | нет         | да               | да           |
| Скоуп                   | глобальный  | по правилам      | по правилам  |
| Скорость                | inline-like | call overhead    | inline-like  |

## Когда уместны функциеподобные макросы

- **Очень короткие** утилиты для генерации кода
- **Условная компиляция** типа `assert`, `DEBUG_PRINT`
- **Header-only "генерики"** в C (где нет templates)
- **Работа с препроцессором** напрямую (string-ification, token-pasting)

В современном C++ для большинства случаев — лучше **`constexpr`-функция**
или **template**.

## Practical relevance для DevOps

- **`assert(x)`** — макрос, который раскрывается в код проверки + abort
- **`LOG(level, msg)`** — добавление файла/строки через `__FILE__` / `__LINE__`
- **`TRY(expr)`** — типичный pattern для обработки ошибок в C-стиле
- **Обёртки над system calls** — единый интерфейс для разных платформ
- **Профилирование:** `MEASURE(name, code)` — обёртка для замера времени
