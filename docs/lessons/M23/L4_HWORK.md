## 1. Продвинутый ввод/вывод дней недели

### Что нужно

Программа: пользователь вводит **порядковый номер** дня недели (1-7),
программа выводит **буквенное** название.

### Ограничения

- **Запрещены классические переменные** (кроме хранения ввода)
- **Запрещены строковые литералы напрямую**
- **Всё через макросы**

### Подсказки

- Token-pasting `##`: `DAY##1` → `DAY1` → раскрывается в макрос со строкой
- Каждый день недели — отдельный макрос: `#define DAY1 "Monday"`
- Макрос-маппер: `#define MAP(n) DAY##n`

## 2. Вывод времени года

### Что нужно

Пользователь — программист. Он определяет один из макросов в самом коде:
`SPRING`, `SUMMER`, `AUTUMN` или `WINTER`.

При компиляции выводится только **выбранный** сезон.

### Ограничения

- Использовать `#if`/`#endif` (условная компиляция)
- Можно обернуть всю `main` функцию

### Подсказки

```cpp
#define SUMMER

#ifdef SUMMER
int main() { std::cout << "Summer\n"; }
#endif
```

При компиляции с `-DSUMMER` или с `#define SUMMER` в коде — выведется именно лето.

## 3. Анализ заполненности вагонов (доп.)

### Что нужно

10 вагонов, в каждом до 20 пассажиров (оптимум).
Пользователь вводит количество пассажиров для каждого вагона.

Программа должна:
1. Сообщить о **переполненных** вагонах (>20)
2. Сообщить о **пустых местах** (<20)
3. Вывести **общее** количество пассажиров

### Ограничения

- **Запрещён for напрямую** — только через макросы (`CALL`-pattern из spec)
- Сами функции — обычные C++, не макросы

### Подсказка из спеки

```cpp
#define CALL(func) { func("Hello, Skillbox!"); }

void print(std::string str) {
    std::cout << str << std::endl;
}

int main() {
    CALL(print);   // → { print("Hello, Skillbox!"); }
}
```

То есть макрос **разворачивается в код**, который вызывает обычную функцию.
