# M15.4 — Сортировка выбором (Selection Sort)

**Длительность:** 12:34
**Источник:** `L4_Сортировка_выбором.vtt` / `.txt`
**Тема:** первый алгоритм сортировки + memory-оптимизация in-place

## Главная идея модуля

> «На каждом шаге выбираем наибольший элемент из оставшихся и кладём в конец.»

Selection Sort — классический O(n²) алгоритм сортировки. Простой, наглядный,
понятный «на пальцах». Учитель показывает 2 версии:
1. **Out-of-place** (доп. вектор результатов) — O(n) extra memory
2. **In-place** через swap — O(1) extra memory

## Версия 1 — Out-of-place

```cpp
#include <vector>
#include <iostream>

int main() {
    std::vector<int> v = {5, -3, 8, 2, -1, 7};
    std::vector<int> res(v.size(), 0);
    for (int i = v.size() - 1; i >= 0; i--) {
        int max_ind = 0;
        for (int j = 0; j < v.size(); j++) {
            if (v[j] > v[max_ind]) max_ind = j;
        }
        res[i] = v[max_ind];
        v[max_ind] = -1;  // «затереть» — wait, это плохо для negative input!
    }
    for (int x : res) std::cout << x << " ";
}
```

**ВНИМАНИЕ:** учитель использует `-1` как sentinel в out-of-place версии,
но в input есть отрицательные числа (`-3`, `-1`). Это **баг** —
если -1 уже был в массиве и его «затёрли», следующая итерация может
найти -1 как максимум (если все остальные тоже отрицательные).

В уроке тест случайно работает потому что там есть положительные числа,
которые «затёрты» -1 не пересекаются с реальными значениями. Но это
ненадёжный паттерн. Правильнее — **INT_MIN** sentinel или **vector<bool> used**.

## Версия 2 — In-place через swap

```cpp
#include <vector>
#include <iostream>

int main() {
    std::vector<int> v = {5, -3, 8, 2, -1, 7};
    for (int i = v.size() - 1; i >= 0; i--) {
        int max_ind = 0;
        for (int j = 0; j <= i; j++) {  // только до i включительно!
            if (v[j] > v[max_ind]) max_ind = j;
        }
        // swap v[i] and v[max_ind]
        int temp = v[i];
        v[i] = v[max_ind];
        v[max_ind] = temp;
    }
    for (int x : v) std::cout << x << " ";
}
```

**Ключевая разница:** второй цикл бежит только до `j <= i`,
потому что элементы `v[i+1..end]` уже отсортированы (последние максимумы).
Это **сужающееся окно поиска**.

## Концепции, которые впервые вводятся

### 1. Сортировка как фундаментальная задача

Учитель явно мотивирует:
- Магазины: сортировка товаров по цене/популярности
- Игры: сортировка футболистов по характеристикам в FIFA
- Системы: почти любое приложение содержит sort где-то

### 2. Selection sort algorithm

Канонический O(n²):
- Внешний цикл: позиция, куда кладём (n итераций)
- Внутренний цикл: поиск max в оставшейся части (n, n-1, n-2, ... = O(n²))
- **Total:** O(n²) compares + O(n) swaps

### 3. Swap pattern с временной переменной

```cpp
int temp = a;
a = b;
b = temp;
```

Базовый swap. Альтернативы которые НЕ упоминаются (но стоит знать):
- `std::swap(a, b)` из `<utility>`
- XOR-swap для int (academic only)
- C++11 move-semantics

### 4. In-place vs out-of-place оптимизация

Это вторая сторона оптимизации (после M15.3 time-оптимизации):
- Time complexity (M15.3): O(n²) → O(n)
- Space complexity (M15.4): O(n) extra → O(1) extra

Selection sort всё равно O(n²) по времени — здесь оптимизировали ТОЛЬКО память.

### 5. Сужающееся окно поиска

`for (j = 0; j <= i; j++)` — невидимый трюк, который превращает
out-of-place в in-place. Зона `[i+1..end]` хранит уже отсортированные
элементы и в неё нельзя «случайно» залезть при поиске следующего max.

## Edge cases (не покрыто в видео)

- Пустой вектор → внешний цикл не запускается, OK
- Один элемент → внешний цикл = 1 итерация, swap самим с собой — OK
- Все равные элементы `{5,5,5}` → max_ind всегда 0, swap=no-op, OK
- Уже отсортированный → лишние swaps, но result правильный
- Обратно отсортированный `{5,4,3,2,1}` → worst case, но result правильный
- Все отрицательные → ВЕРСИЯ 1 (out-of-place) **сломается** из-за -1 sentinel

Эти edge cases надо покрыть в hwork-тестах.

## Алгоритмическая характеристика

| Метрика     | Selection Sort  |
|-------------|-----------------|
| Time best   | O(n²)           |
| Time avg    | O(n²)           |
| Time worst  | O(n²)           |
| Space       | O(1) (in-place) |
| Stable      | NO              |
| Adaptive    | NO              |

**Stable** = равные элементы сохраняют относительный порядок.
Selection sort не stable: при swap с дальним max можно перепрыгнуть равный элемент.

**Adaptive** = быстрее на «почти отсортированных» данных.
Selection sort всегда O(n²) — не адаптивен.

| Bubble       | O(n)     | O(n²)    | O(n²)    | O(1)    | Yes    |
| Insertion    | O(n)     | O(n²)    | O(n²)    | O(1)    | Yes    |
| Merge        | O(nlogn) | O(nlogn) | O(nlogn) | O(n)    | Yes    |

## Что НЕ показано но стоит знать

1. **`std::sort()`** — встроенный sort, O(n log n) avg
2. **`std::stable_sort()`** — стабильная версия, O(n log² n)
3. **Кастомный compare** через лямбды:
   ```cpp
   std::sort(v.begin(), v.end(), [](int a, int b) { return a > b; }); // descending
   ```
4. **Sort by key** (например структуры по полю):
   ```cpp
   std::sort(items.begin(), items.end(),
             [](const Item& a, const Item& b) { return a.price < b.price; });
   ```

Эти инструменты будут гораздо более продуктивны в реальном коде, но
понимание базовых сортировок необходимо для интервью и базового CS.
