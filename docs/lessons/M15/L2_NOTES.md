## Задача 1 — Количество уникальных элементов

**Вход:** `vector<int> v = {1, 1, 2, 5, 5, 5, 3, 3, 8}`
**Выход:** `5` (уникальные: 1, 2, 5, 3, 8)

### Подход «вычёркивания» (sentinel = -1)

```cpp
#include <vector>
#include <iostream>

int main() {
    std::vector<int> v = {1, 1, 2, 5, 5, 5, 3, 3, 8};
    for (int i = 0; i < v.size(); i++) {
        if (v[i] == -1) continue;          // уже «вычеркнут»
        for (int j = i + 1; j < v.size(); j++) {
            if (v[j] == v[i]) v[j] = -1;   // вычёркиваем дубль
        }
    }
    int res = 0;
    for (int x : v) if (x != -1) res++;
    std::cout << res << std::endl;          // → 5
}
```

**Ключ:** -1 как sentinel «обработан». Работает только если **все элементы >= 0**.

## Задача 2 — Наиболее часто встречающийся элемент

**Вход:** `{1, 2, 2, 3, 3, 3, 4, 4, 4, 4, 5}`
**Выход:** `4` (встречается 4 раза)

### Подход: temp[i] = доп. встречи v[i] до конца + max-index

```cpp
#include <vector>
#include <iostream>

int main() {
    std::vector<int> v = {1, 2, 2, 3, 3, 3, 4, 4, 4, 4, 5};
    std::vector<int> temp(v.size(), 0);    // size + init value!
    for (int i = 0; i < v.size(); i++) {
        if (v[i] == -1) continue;
        for (int j = i + 1; j < v.size(); j++) {
            if (v[j] == v[i]) {
                v[j] = -1;
                temp[i]++;                  // считаем встречи
            }
        }
    }
    int max_ind = 0;
    for (int i = 0; i < temp.size(); i++) {
        if (temp[i] > temp[max_ind]) max_ind = i;
    }
    std::cout << v[max_ind] << std::endl;   // → 4
}
```

**Хитрость:** `v[max_ind]` уже исходное значение, не -1, потому что
«первый встреченный» элемент НЕ помечается -1 (только дубли после него).

## Концепции, которые впервые показывают

### 1. `vector<T>(size, value)` — двухаргументный конструктор

```cpp
std::vector<int> temp(v.size(), 0);  // size элементов, все = 0
std::vector<int> a(10, -1);          // 10 минус-единичек
```

Раньше показывали только resize/push_back/{...} list-init. Это удобнее
когда нужен массив нужного размера с «нейтральным» значением.

### 2. Sentinel-значения для «обработано»

`-1` маркирует «вычеркнут» (если домен >= 0). Альтернативы:
- `INT_MIN` если входят отрицательные
- параллельный `vector<bool> used` (более чистый, но +память)
- swap-and-pop в конец массива (in-place compaction, без sentinel)

### 3. Max-index pattern

```cpp
int max_ind = 0;
for (int i = 1; i < v.size(); i++) {
    if (v[i] > v[max_ind]) max_ind = i;
}
// теперь и max-value (v[max_ind]) и индекс известны
```

Когда нужны оба — индекс и значение — лучше вести индекс.
Из M14.3 (matrix×vector) уже видели аналогичный паттерн с tail-индексом.

### 4. Reuse предыдущего решения

Task 2 явно строится на Task 1 («я не удалял код из предыдущей задачи»).
Учитель показывает: один паттерн «вычёркивания» решает оба класса задач —
надо только дополнить его счётчиком в `temp[]`.

## Edge cases (что НЕ покрыто в видео)

- Отрицательные числа во входе → -1 sentinel ломается
- Пустой вектор → `max_ind = 0` обращается к v[0] = UB
- Все элементы одинаковые → корректно (temp[0] = size-1, max = v[0])
- Все элементы уникальны → temp всё нули, max_ind = 0 (любой ответ ок)
- Несколько элементов с одинаковой частотой → возвращается первый (стабильно)

Эти edge cases стоит закрыть в hwork-тестах (см. M15.6).

## Алгоритмическая характеристика

- Time: O(n²) — два вложенных цикла
- Space: O(n) для temp (Task 2)
- Trade-off: можно O(n log n) через сортировку, O(n) через hash-map.
  Учитель сознательно не идёт туда — паттерн «как на листе» проще для intro.
