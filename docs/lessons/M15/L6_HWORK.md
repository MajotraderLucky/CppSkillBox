## Задача 1 — Maximum Subarray (LeetCode 53)

### Что нужно сделать

Дан массив целых чисел. Найти такие индексы `i` и `j`, что
`a[i] + a[i+1] + ... + a[j]` будет максимально возможной. Вывести `i` и `j`.

### Пример

```
Input:  a = {-2, 1, -3, 4, -1, 2, 1, -5, 4}
Output: 3 6
```

Наибольшая сумма последовательных элементов между индексами 3 и 6:
`{4, -1, 2, 1}`, сумма = 6.

### Algorithmic note

- Naive: O(n³) — два цикла + sum
- Better: O(n²) — сохранить prefix sums
- Optimal: O(n) — **Kadane's algorithm**

Учитель в M15.3 показывал single-pass с tracking минимума — здесь похожий
паттерн с tracking максимума «текущей» суммы. Kadane = классика.

## Задача 2 — Two Sum (LeetCode 1)

### Что нужно сделать

Дан массив целых чисел + одно target число. Найти **2 числа** в массиве,
сумма которых = target. Гарантируется ровно одна пара. Вывести числа.

### Пример

```
Input:  a = {2, 7, 11, 15}, target = 9
Output: 2 7
```

(2 + 7 = 9)

### Рекомендация

Как только нашли первую пару — **early exit** (break).

### Algorithmic note

- Naive: O(n²) — два вложенных цикла (учитель скорее всего ждёт это)
- Optimal: O(n) — hash-map (`std::unordered_map<int, int>` value→index)
  - но `unordered_map` ещё не проходили в курсе → использовать наивный

## Задача 3 — Top-5 smallest (online algorithm)

### Что нужно сделать

С клавиатуры вводятся числа. Когда пользователь:
- вводит `-1`: вывести 5-е по возрастанию из ВСЕХ введённых
- вводит `-2`: завершить программу

### Пример

```
Ввод: 7 5 3 1 2 4 6 -1
Вывод: 5 (отсортированно: {1, 2, 3, 4, 5, 6, 7})

Ввод: 1 1 1 -1
Вывод: 2 (после прошлого ввода уже есть {1,2,3,4,5,6,7}, добавили 1 1 1)

Ввод: -2
Завершение программы
```

### Важная оптимизация

> **Важная часть задания — оптимальное использование ресурсов.**
> Для решения этой задачи достаточно хранить в векторе **только пять наименьших элементов**.

Если уже есть `[1,3,5,7,9]` и приходит `15` — отбрасываем (никогда не будет в top-5).
Если приходит `8` — заменяем `9` на `8` → `[1,3,5,7,8]`.

### Рекомендация

Удостоверьтесь что число **можно вывести** — может быть < 5 чисел!

### Algorithmic note

- Naive: хранить всё, sort при `-1` запросе → O(n log n) per query, O(n) memory
- Optimal: maintain top-5 only → O(n) total, O(1) memory (constant 5)
  - При новом элементе: если меньше max из top-5 → заменить max и пересортировать top-5
  - Это **partial sort / order statistic** в миниатюре

В индустрии это решается через **min-heap of size 5** (приоритетная очередь),
но heap ещё не проходили. Обойдёмся sorted vector.

## Задача 4 — Sort by absolute value (преобразование отсортированного)

### Что нужно сделать

Дан массив целых чисел, **отсортированный по возрастанию**.
Вывести в порядке возрастания **модуля**.

### Пример

```
Input:  {-100, -50, -5, 1, 10, 15}
Output: 1 -5 10 15 -50 -100
```

### Рекомендация

> Решение этой задачи **не требует новой сортировки** массива, так как массив уже отсортирован.
> Нужно лишь правильно организовать проход по массиву с выдачей значений по условию задачи.

### Подсказка

> Можно двигаться **от первого положительного элемента** влево или вправо,
> выбирая следующий элемент, больший по модулю.

### Algorithmic note

Это **two-pointer / merge** pattern:
- Найти границу negative/positive (e.g. binary search → O(log n), или linear → O(n))
- Два указателя: `left` идёт назад от границы (negative side), `right` идёт вперёд (positive side)
- На каждом шаге берём тот, чей abs меньше
- Total: O(n) time, O(n) output (просто вывод)

Аналогия с **merge step** в merge sort.

LeetCode эквивалент: [977. Squares of Sorted Array](https://leetcode.com/problems/squares-of-a-sorted-array/) (с небольшим отличием — там square, тут abs, но идея идентична).
