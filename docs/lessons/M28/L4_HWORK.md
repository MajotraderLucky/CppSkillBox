## 1. Заплыв на 100 метров

### Что нужно

- 6 пловцов, 6 дорожек
- Имена + скорости (м/с) — из stdin
- Каждый пловец = отдельный поток
- Каждую секунду — вывод текущего расстояния
- Когда все доплыли — таблица результатов сортированная по возрастанию времени

### Подсказки

- `std::thread` для каждого пловца
- `sleep_for(1s)` симулирует «секунду»
- mutex для общего вывода (чтобы строки не путались)

---

## 2. Симуляция вокзала

### Что нужно

3 поезда (A/B/C) с разным временем в пути.
Один вокзал — единовременно только 1 поезд.

Для каждого поезда:
1. Едет N сек
2. Прибывает (если занято — ждёт)
3. Ждёт команду `depart` от пользователя
4. Уезжает

Завершение — все 3 поезда побывали на вокзале.

### Подсказки

- `std::mutex stationMutex` — захват = поезд на вокзале
- При прибытии: `lock()` (ждёт если занято)
- При отбытии: `unlock()`
- Ждать команды `depart` через std::cin под отдельным mutex

## 3. Кухня онлайн-ресторана

### Что нужно

- **Заказы:** каждые 5-10 сек поступает случайное блюдо (5 типов: пицца/суп/стейк/салат/суши)
- **Кухня:** 5-15 сек готовит (только одна готовка одновременно)
- **Курьер:** каждые 30 сек забирает все готовые
- **Завершение:** после 10 успешных доставок

### Подсказки

- `std::mutex kitchenMutex` — для готовки
- `detach()` для готовки (fire-and-forget)
- mutex для очередей (заказов, готовых)

## Подход к тестированию

Многопоточные программы сложно детерминированно тестировать.
Стратегия:
1. CLI-флаги для seed/timing scale (детерминизм)
2. Проверка инвариантов вместо точного output
3. Проверка что все потоки завершились (нет hang)
4. Использовать `--no-sleep` или `--scale=10` для ускорения тестов
