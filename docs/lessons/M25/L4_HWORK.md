## 1. Симулятор хирургической операции

### Что нужно

Программа-симулятор медицинской операции. Команды:

| Команда     | Аргументы               | Действие                          |
|-------------|-------------------------|-----------------------------------|
| `scalpel`   | x1 y1 x2 y2 (две точки) | разрез между точками              |
| `hemostat`  | x y (одна точка)        | зажим в точке                     |
| `tweezers`  | x y (одна точка)        | пинцет в точке                    |
| `suture`    | x1 y1 x2 y2 (две точки) | хирургический шов между точками   |

### Правила

- Операция начинается **с `scalpel`** и заканчивается **с `suture`**
- Выход: когда **точки `suture` совпадают**
- Координаты — `double` X, Y

### Структура проекта

```text
hwork25_1/
├── CMakeLists.txt
├── include/
│   └── tools.h        ← функции-инструменты
└── src/
    ├── main.cpp       ← основная логика, обработка ввода
    └── tools.cpp      ← реализации инструментов
```

### Подсказки

- Структура `Point2D { double x, y; }` + функции `read`, `print`, `equal`
- Spec говорит «одного заголовочного и одного CPP» — на самом деле main + tools.h + tools.cpp

## 2. Симулятор работы компьютера (multi-module)

### Что нужно

5 компонент, каждая в **отдельном модуле**:

| Компонент | Команда       | Функция                                                    |
|-----------|---------------|------------------------------------------------------------|
| `cpu`     | `sum`         | сложить 8 чисел из RAM, вывести результат                  |
| `ram`     | (внутр.)      | буфер 8 чисел + read/write API                             |
| `disk`    | `save`/`load` | сохранить/загрузить 8 чисел в/из data.txt                  |
| `gpu`     | `display`     | вывести 8 чисел из RAM в консоль                           |
| `kbd`     | `input`       | ввести 8 чисел и записать в RAM                            |

`exit` — выход.

### Жёсткие требования

- **Доступ к буферу RAM только через модуль ram** — другие модули используют API
- **2 CMakeLists.txt** — root + `src/CMakeLists.txt`
- **`#include` без указания папок** — `#include "ram.h"`, не `#include "../include/ram.h"`
- **Header'ы только с объявлениями**, реализации в .cpp

### Структура

```text
hwork25_2/
├── CMakeLists.txt          ← root
├── include/
│   ├── cpu.h
│   ├── ram.h
│   ├── disk.h
│   ├── gpu.h
│   └── kbd.h
└── src/
    ├── CMakeLists.txt      ← вложенный
    ├── main.cpp
    ├── cpu.cpp
    ├── ram.cpp
    ├── disk.cpp
    ├── gpu.cpp
    └── kbd.cpp
```

### Подсказки

- `#pragma once` в каждом header'е
- Дотошно подумать о dependency между модулями (cpu зависит от ram; gpu зависит от ram, disk зависит от ram)
- В CMake: `target_include_directories(target PUBLIC ${CMAKE_SOURCE_DIR}/include)`
