# M16.1 — О чём этот модуль (longread overview)

**Тип:** longread (без видео)
**URL:** `longread/a57a8061-b828-4b49-9ca1-ca9173687ba3`

## Что рассмотрим в модуле

1. **Фундаментальная задача преобразования данных** из одного формата в другой:
   - игра: mouse/keyboard → картинка
   - таблицы: числа → строки/столбцы → графики
   - видеоплеер: 0/1 → видеоряд

2. **Внутреннее устройство форматов:** как хранятся данные в разных форматах
   и какие готовые механизмы для автоматической обработки существуют.

3. **Базовые типы данных и их внутреннее устройство:**
   - `char`, `int`, `float`, `string`
   - размещение массивов в памяти

4. **Преобразование чисел в строки и обратно** (готовые функции stoi/to_string и т.д.)

5. **Перечисления (enum)** — именованные константы, в т.ч. как bit flags
   для хранения «группы переключателей» в одной integer переменной.

## Что будет после модуля

- Уметь преобразовывать данные string ↔ numeric
- Знать как примитивные типы устроены внутри (для отладки и понимания edge cases)
- Понимать почему некоторые ошибки происходят (overflow, precision loss, encoding)
- Использовать enum + bitmask для compact state representation

## Связь с другими модулями

- **M2 (numeric types):** этот модуль раскрывает что было «под капотом»
- **M9 (strings):** дополняет конверсиями между string и numeric
- **M17+:** видимо классы/struct (типы as data carrier) — лестница вверх

## Practical importance

**HIGH.** Это одна из тех тем что DevOps/SRE используют каждый день:
- парсинг логов (string → struct)
- сериализация конфигов (struct → JSON/YAML/TOML)
- битовая работа с флагами (Linux file modes, signal flags, etc.)
