# M18.2 — Ссылки (references)

**Длительность:** ~5 минут
**Тема:** references — псевдонимы переменных, const-ссылки

## Главная идея

> **Ссылка = псевдоним для существующей переменной.** Изменение через ссылку = изменение оригинала.

## Объявление и использование

```cpp
int i = 10;
int& j = i;       // j — псевдоним для i

j = 20;           // меняет i (т.к. j это i)
std::cout << i;   // 20
```

**Два правила создания:**
1. После типа — символ `&` (отличается от & как «взятие адреса»)
2. Сразу инициализируется — нельзя создать «пустую» ссылку

## Const-ссылки

```cpp
int i = 10;
const int& j = i;   // const-ссылка

j = 20;             // ERROR: cannot assign to const
i = 20;             // OK: оригинал по-прежнему mutable
std::cout << j;     // 20 (читаем через const-ссылку)
```

Паттерн «смотри, но не трогай» — позволяет дать read-only доступ.

## `&i` vs `&j` — одинаковые адреса

```cpp
int i = 10;
int& j = i;
std::cout << &i;  // 0x7ffff...
std::cout << &j;  // тот же адрес
```

Ссылка НЕ имеет своего адреса — она лишь алиас. Это отличает её от указателя:
- Pointer: имеет свой адрес (`&p`), может перенаправляться (`p = &other`)
- Reference: алиас, **никогда не меняет привязку**, нет своего адреса

## Pointer vs Reference

| Аспект          | Pointer                 | Reference                |
|-----------------|-------------------------|--------------------------|
| Объявление      | `int* p = &i;`          | `int& r = i;`            |
| Дереференс      | `*p`                    | `r` (автоматически)      |
| Re-assignment   | `p = &other;` ОК        | НЕЛЬЗЯ — связь fixed     |
| Может быть NULL | `int* p = nullptr;` ОК  | НЕТ — должен быть init   |
| Свой адрес      | Да: `&p`                | НЕТ                      |
| Арифметика      | `p++` OK                | НЕТ                      |

## Practical relevance для DevOps

- **Function parameters by reference:** избегаем копирования больших объектов
- **Iteration:** `for (auto& item : container)` (modify in place)
- **`const std::string&` в API:** идиома для read-only string passing

## Связь с другими модулями
